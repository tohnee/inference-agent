## 📐 Skill 1：正确度量——CUDA Event 异步计时基础设施

在做任何优化之前，必须先建立**不干扰热路径**的计时系统。用 `time.time()` 或 `torch.cuda.synchronize()` 测 GPU 延迟是最常见的错误。

### 核心原则

CUDA kernel 启动是**异步**的——CPU 提交命令后立即返回，GPU 在队列里慢慢执行。如果在热路径中插入 `synchronize()`，会强制 CPU 等待 GPU 排空队列，测出来的数字既不准确也不可复现。[2]

### 生产级计时模板

```python
import torch
import numpy as np
from contextlib import contextmanager
from dataclasses import dataclass, field
from typing import Dict, List

@dataclass
class StageTimer:
    """单阶段 CUDA Event 计时器，完全异步，不阻塞热路径。"""
    name: str
    starts: List[torch.cuda.Event] = field(default_factory=list)
    ends:   List[torch.cuda.Event] = field(default_factory=list)

    def tick(self):
        e = torch.cuda.Event(enable_timing=True)
        e.record()
        self.starts.append(e)

    def tock(self):
        e = torch.cuda.Event(enable_timing=True)
        e.record()
        self.ends.append(e)

    def summarize(self) -> dict:
        # 所有 elapsed_time 在 summary 阶段一次性 synchronize
        torch.cuda.synchronize()
        times = [s.elapsed_time(e) for s, e in zip(self.starts, self.ends)]
        return {
            "name":  self.name,
            "mean":  np.mean(times),
            "p50":   np.percentile(times, 50),
            "p95":   np.percentile(times, 95),
            "min":   np.min(times),
            "count": len(times),
        }


class ProfilerContext:
    """多阶段 profiling 上下文，支持 warm-up 自动跳过。"""

    def __init__(self, warmup: int = 3):
        self._timers: Dict[str, StageTimer] = {}
        self._warmup = warmup
        self._call_count = 0

    @contextmanager
    def stage(self, name: str):
        if name not in self._timers:
            self._timers[name] = StageTimer(name)
        timer = self._timers[name]
        skip = self._call_count < self._warmup
        if not skip:
            timer.tick()
        yield
        if not skip:
            timer.tock()

    def next_call(self):
        self._call_count += 1

    def report(self):
        print(f"\n{'─'*72}")
        print(f"  {'Stage':<28} {'Mean':>8} {'P50':>8} {'P95':>8} {'Min':>8} {'N':>5}")
        print(f"{'─'*72}")
        for name, timer in self._timers.items():
            s = timer.summarize()
            print(f"  {s['name']:<28} {s['mean']:>7.2f}ms {s['p50']:>7.2f}ms "
                  f"{s['p95']:>7.2f}ms {s['min']:>7.2f}ms {s['count']:>5}")
        print(f"{'─'*72}\n")
```

### 关键判断标准

| 计时方法 | 适用场景 | 误差来源 |
|---|---|---|
| `torch.cuda.Event` | GPU kernel 计时（推荐） | 几乎无误差 |
| `time.perf_counter` | CPU 端逻辑计时 | 不含 GPU 执行时间 |
| `time.time` + `synchronize` | ❌ 热路径禁用 | 强制串行化，破坏流水线 |
| `torch.profiler` | 详细 trace 分析（非热路径） | overhead 较大 |

## 🔍 Skill 2：系统性 Profiling——找到真正的瓶颈

Profiling 的目标不是"找到慢的地方"，而是**找到主要矛盾**。文档中最关键的洞察是：脚本自动建议的 P1 优化（DataLoader workers）在真实业务场景下几乎没有价值，因为它优化的是一个不存在的场景。

### 完整 Profiling 脚本模板

```python
import torch, hashlib, os
import numpy as np
from pathlib import Path

def run_full_profile(
    compute_fn,          # 被测函数
    query_arr,
    gallery_arr,
    n_warmup: int = 3,
    n_repeat: int = 10,
):
    """
    分离 Cold Run / Warm Run，输出 CPU-GPU Overlap 分析。
    """
    profiler = ProfilerContext(warmup=n_warmup)

    print("=== Phase 1: Cold Run (Gallery 未缓存) ===")
    cold_times = []
    for i in range(n_warmup + n_repeat):
        profiler.next_call()
        t0 = torch.cuda.Event(enable_timing=True)
        t1 = torch.cuda.Event(enable_timing=True)
        t0.record()
        with profiler.stage("cold_total"):
            result = compute_fn(query_arr, gallery_arr, use_cache=False)
        t1.record()
        torch.cuda.synchronize()
        if i >= n_warmup:
            cold_times.append(t0.elapsed_time(t1))

    print("=== Phase 2: Warm Run (Gallery 已缓存) ===")
    warm_times = []
    for i in range(n_warmup + n_repeat):
        t0 = torch.cuda.Event(enable_timing=True)
        t1 = torch.cuda.Event(enable_timing=True)
        t0.record()
        result = compute_fn(query_arr, gallery_arr, use_cache=True)
        t1.record()
        torch.cuda.synchronize()
        if i >= n_warmup:
            warm_times.append(t0.elapsed_time(t1))

    # CPU-GPU Overlap 分析
    _print_overlap_analysis(cold_times, warm_times)
    profiler.report()
    return result


def _print_overlap_analysis(cold_times, warm_times):
    print(f"\n{'═'*60}")
    print(f"  Cold Run  mean={np.mean(cold_times):.1f}ms  "
          f"p95={np.percentile(cold_times,95):.1f}ms")
    print(f"  Warm Run  mean={np.mean(warm_times):.1f}ms  "
          f"p95={np.percentile(warm_times,95):.1f}ms")
    print(f"  Cache speedup: {np.mean(cold_times)/np.mean(warm_times):.1f}×")
    print(f"{'═'*60}\n")
```

### Profiling 结果解读框架

拿到 profiling 数据后，按以下顺序分析：

```
Step 1: 区分 Cold Run vs Warm Run
        → 如果业务是"Gallery 固定，Query 频繁"，
          Warm Run 才是核心指标

Step 2: 找 CPU-GPU Overlap
        → preprocess_time > inference_time？
          → CPU 是瓶颈，增加 num_workers 收益有限
        → H2D_time > 5% of inference_time？
          → 考虑 uint8 传输或 GPU 预处理

Step 3: 识别"假瓶颈"
        → 每次都重建 Gallery？→ 缓存，不是优化推理
        → sim_matrix 在 CPU？→ 移到 GPU，2000× 加速
        → batch=1？→ padding，不是优化模型

Step 4: 估算优化上限
        → 物理极限：PCIe H2D 带宽（L20 约 16GB/s）
        → 不要优化到物理极限以下
```

## ⚡ Skill 3：算子级优化——SDPA Backend 选择与编译策略

### 3.1 SDPA Backend 基准测试（不靠猜）

PyTorch 的 `scaled_dot_product_attention` 会自动选择 backend，但"自动"不等于"最优"。对于 ViT 这类**短序列**模型，Memory Efficient 通常优于 Flash Attention。[1]

```python
import torch, contextlib, numpy as np
from transformers import AutoModel

def benchmark_sdpa_backends(
    model_name: str,
    batch_size: int = 64,
    img_size: int = 256,
    n_warmup: int = 10,
    n_repeat: int = 50,
    device: str = "cuda",
):
    """
    实测三种 SDPA backend，返回排序结果。
    必须在目标 GPU 上运行——不同硬件结果不同。
    """
    device = torch.device(device)
    model = AutoModel.from_pretrained(model_name).to(device).eval()
    dummy = torch.randn(batch_size, 3, img_size, img_size,
                        device=device, dtype=torch.float32)

    backends = [
        ("Flash Attention",    True,  False, False),
        ("Memory Efficient",   False, True,  False),
        ("Math SDPA",          False, False, True),
        ("Flash + MemEff",     True,  True,  False),
    ]

    results = []
    for name, flash, mem_eff, math in backends:
        # 设置 backend
        torch.backends.cuda.enable_flash_sdp(flash)
        torch.backends.cuda.enable_mem_efficient_sdp(mem_eff)
        torch.backends.cuda.enable_math_sdp(math)

        # Warm-up
        with torch.no_grad():
            for _ in range(n_warmup):
                _ = model(dummy).last_hidden_state
        torch.cuda.synchronize()

        # 计时
        starts = [torch.cuda.Event(enable_timing=True) for _ in range(n_repeat)]
        ends   = [torch.cuda.Event(enable_timing=True) for _ in range(n_repeat)]
        with torch.no_grad():
            for i in range(n_repeat):
                starts[i].record()
                _ = model(dummy).last_hidden_state
                ends[i].record()
        torch.cuda.synchronize()

        times = [s.elapsed_time(e) for s, e in zip(starts, ends)]
        results.append({
            "name": name, "mean": np.mean(times),
            "p95": np.percentile(times, 95), "min": np.min(times)
        })

    # 恢复默认
    torch.backends.cuda.enable_flash_sdp(True)
    torch.backends.cuda.enable_mem_efficient_sdp(True)
    torch.backends.cuda.enable_math_sdp(True)

    # 排序输出
    results.sort(key=lambda x: x["mean"])
    print(f"\nSDPA Backend Benchmark — {model_name} | batch={batch_size}")
    print(f"{'Backend':<22} {'Mean':>8} {'P95':>8} {'Min':>8}")
    print("─" * 50)
    for r in results:
        marker = " ← 最优" if r == results[0] else ""
        print(f"  {r['name']:<20} {r['mean']:>7.2f}ms "
              f"{r['p95']:>7.2f}ms {r['min']:>7.2f}ms{marker}")

    best = results[0]["name"]
    print(f"\n建议写入 CLAUDE.md：SDPA backend = {best}")
    return results
```

### 3.3 torch.compile 策略与 Graph Break 检测

在用 `torch.compile` 之前，**先检测 graph break 数量**，再决定是否值得投入。[3] [4]

```python
import torch
import logging

def detect_graph_breaks(model_fn, example_input, verbose: bool = True):
    """
    检测 graph break 数量，评估 fullgraph=True 的可行性。
    返回 (break_count, compiled_fn)
    """
    if verbose:
        torch._logging.set_logs(graph_breaks=True)

    break_count = 0
    original_warn = torch._dynamo.utils.counters

    # 用 fullgraph=False 先统计 break 数量
    compiled = torch.compile(
        model_fn,
        mode="reduce-overhead",
        fullgraph=False,
        dynamic=False,
    )

    with torch.no_grad():
        _ = compiled(example_input)

    # 读取 dynamo 计数器
    breaks = torch._dynamo.utils.counters.get("graph_break", {})
    break_count = sum(breaks.values())

    print(f"\nGraph Break 分析:")
    print(f"  总计: {break_count} 处 graph break")

    if break_count == 0:
        print("  ✅ 可以安全使用 fullgraph=True，预期额外提升 5-15%")
        recommendation = "fullgraph=True"
    elif break_count <= 3:
        print("  ⚠️  少量 graph break，评估修复成本")
        recommendation = "评估后决定"
    else:
        print(f"  ❌ {break_count} 处 graph break，fullgraph 不可行")
        print("     建议：使用 mode='reduce-overhead' 替代")
        recommendation = "reduce-overhead"

    print(f"  建议模式: {recommendation}")

    if verbose:
        for reason, count in breaks.items():
            print(f"    - {reason}: {count} 次")

    return break_count, compiled


def get_compile_config(break_count: int) -> dict:
    """根据 graph break 数量返回最优编译配置。"""
    if break_count == 0:
        return {
            "mode": "max-autotune",
            "fullgraph": True,
            "dynamic": False,
        }
    elif break_count <= 3:
        return {
            "mode": "max-autotune",
            "fullgraph": False,
            "dynamic": False,
        }
    else:
        return {
            "mode": "reduce-overhead",
            "fullgraph": False,
            "dynamic": True,   # graph break 多时用 dynamic 减少重编译
        }
```

## 🗄️ Skill 4：缓存策略——内存缓存 + 持久化缓存双层架构

这是文档中收益最大的优化方向。Gallery 缓存把 Warm Run 从 13852ms 降到 42ms，是纯技术优化无法企及的数量级提升。

### 双层缓存架构

```python
import hashlib, threading, torch
import numpy as np
from pathlib import Path
from typing import Optional

class TwoLevelGalleryCache:
    """
    双层 Gallery 特征缓存：
    L1 = 内存缓存（进程内，毫秒级命中）
    L2 = 磁盘缓存（跨重启，秒级命中）
    """

    def __init__(self, cache_dir: str = "./gallery_cache"):
        self._lock     = threading.RLock()
        self._l1_key   = None
        self._l1_feats = None          # CPU tensor，不占 GPU 显存
        self._cache_dir = Path(cache_dir)
        self._cache_dir.mkdir(parents=True, exist_ok=True)

    # ── 哈希计算 ──────────────────────────────────────────────
    def _hash(self, arr: np.ndarray) -> str:
        """用内容 MD5 作为 key，Gallery 更新自动失效。"""
        return hashlib.md5(arr.tobytes()).hexdigest()[:16]

    def _disk_path(self, h: str, n: int) -> Path:
        return self._cache_dir / f"gallery_{h}_n{n}.pt"

    # ── L1 内存缓存 ───────────────────────────────────────────
    def _l1_get(self, h: str) -> Optional[torch.Tensor]:
        with self._lock:
            if self._l1_key == h:
                return self._l1_feats
        return None

    def _l1_set(self, h: str, feats: torch.Tensor):
        with self._lock:
            self._l1_key   = h
            # 存 FP32 CPU tensor，避免长期占用 GPU 显存
            self._l1_feats = feats.float().cpu()

    # ── L2 磁盘缓存 ───────────────────────────────────────────
    def _l2_get(self, h: str, n: int) -> Optional[torch.Tensor]:
        path = self._disk_path(h, n)
        if path.exists():
            return torch.load(path, map_location="cpu", weights_only=True)
        return None

    def _l2_set(self, h: str, n: int, feats: torch.Tensor):
        path = self._disk_path(h, n)
        torch.save(feats.float().cpu(), path)

    # ── 主接口 ────────────────────────────────────────────────
    def get_or_build(
        self,
        gallery_arr: np.ndarray,
        build_fn,              # callable: (gallery_arr) -> torch.Tensor
        device: torch.device,
    ) -> torch.Tensor:
        h = self._hash(gallery_arr)
        n = len(gallery_arr)

        # L1 命中
        feats = self._l1_get(h)
        if feats is not None:
            print(f"[Cache L1 HIT]  {n} 张图，跳过推理")
            return feats.to(device)

        # L2 命中
        feats = self._l2_get(h, n)
        if feats is not None:
            print(f"[Cache L2 HIT]  从磁盘加载 {n} 张图特征")
            self._l1_set(h, feats)           # 回填 L1
            return feats.to(device)

        # Cache MISS：重新推理
        print(f"[Cache MISS]    推理 {n} 张图...")
        feats = build_fn(gallery_arr)        # GPU tensor

        self._l1_set(h, feats)               # 写 L1
        self._l2_set(h, n, feats)            # 写 L2
        print(f"[Cache SAVED]   → {self._disk_path(h, n)}")
        return feats

    def invalidate(self):
        """手动失效（Gallery 强制更新时调用）。"""
        with self._lock:
            self._l1_key   = None
            self._l1_feats = None
        print("[Cache INVALIDATED] L1 已清空，L2 保留（内容变化自动失效）")

    def stats(self) -> dict:
        disk_files = list(self._cache_dir.glob("gallery_*.pt"))
        total_mb   = sum(f.stat().st_size for f in disk_files) / 1e6
        return {
            "l1_loaded": self._l1_key is not None,
            "l2_files":  len(disk_files),
            "l2_total_mb": round(total_mb, 1),
        }


# 全局单例
GALLERY_CACHE = TwoLevelGalleryCache()
```

## 🚀 Skill 5：GPU 利用率优化——Query Padding + GPU 预处理流水线

### 5.1 Query Padding：消灭 batch=1 的 kernel launch overhead

batch=1 时 GPU 利用率约 1.5%，padding 到满批次是最低成本的 GPU 利用率优化。[3]

```python
import torch
import torch.nn.functional as F
import numpy as np

def infer_single_with_padding(
    backbone: torch.nn.Module,
    query_tensor: torch.Tensor,    # (1, C, H, W)
    pad_to_batch: int = 64,
    device: torch.device = torch.device("cuda"),
    pooling: str = "gap+cls",      # "gap+cls" | "cls" | "gap"
) -> torch.Tensor:
    """
    将单张 Query padding 到满批次推理，提升 GPU 利用率。
    只返回第 0 个结果，padding 部分丢弃。

    性能参考（ViT-B/16, L20）：
      batch=1  → 6.47ms/query（GPU 利用率 ~1.5%）
      batch=64 → 1.02ms/query（GPU 利用率 ~96%）
    """
    assert query_tensor.shape[0] == 1, "此函数专为单张 Query 设计"

    # expand 不复制数据（共享内存），零额外显存开销
    padded = query_tensor.expand(pad_to_batch, -1, -1, -1).to(device)

    with torch.no_grad():
        out = backbone(padded)

    hidden = out.last_hidden_state  # (B, seq_len, D)

    # 特征池化
    if pooling == "gap+cls":
        cls_feat = hidden[:, 0, :]           # CLS token
        gap_feat = hidden[:, 1:, :].mean(1)  # Global Average Pooling
        feat = torch.cat([cls_feat, gap_feat], dim=-1)
    elif pooling == "cls":
        feat = hidden[:, 0, :]
    elif pooling == "gap":
        feat = hidden[:, 1:, :].mean(1)
    else:
        raise ValueError(f"Unknown pooling: {pooling}")

    return feat[0:1]   # (1, D)，只取第一个结果
```

### 5.2 GPU 预处理流水线：uint8 H2D + GPU resize/normalize

```python
import torch
import torchvision.transforms.v2 as T
import numpy as np

# 预构建 GPU 变换（模块级单例，避免重复初始化）
_GPU_TRANSFORM_256 = T.Compose([
    T.Resize((256, 256), antialias=True),
    T.ToDtype(torch.float32, scale=True),     # uint8→float32，[0,255]→[0,1]
    T.Normalize(
        mean=[0.485, 0.456, 0.406],
        std =[0.229, 0.224, 0.225]
    ),
])

def preprocess_on_gpu(
    img_uint8: np.ndarray,          # (H, W, 3) uint8
    device: torch.device,
    transform=_GPU_TRANSFORM_256,
) -> torch.Tensor:
    """
    GPU 预处理流水线。
    H2D 传输 uint8（0.8MB）而非 float32（10MB），节省 ~18ms PCIe 时间。

    传输对比（L20, PCIe 16GB/s）：
      float32 (H,W,3) 256×256 → 0.75MB × 4 = 3MB → ~0.19ms（理论）
      实际含 CUDA stream overhead：~20ms
      uint8  (H,W,3) 256×256 → 0.75MB × 1 = 0.75MB → ~0.05ms（理论）
      实际：~1.6ms
    """
    # HWC uint8 → CHW uint8
    t = torch.from_numpy(img_uint8).permute(2, 0, 1)   # (3, H, W) uint8

    # H2D：non_blocking=True，不阻塞 CPU
    t = t.to(device, non_blocking=True)                 # 0.75MB uint8

    # GPU 上完成 resize + normalize
    t = transform(t.unsqueeze(0))                       # (1, 3, 256, 256) float32

    return t


def compute_similarity_gpu(
    query_feat: torch.Tensor,    # (1, D)
    gallery_feat: torch.Tensor,  # (N, D)
) -> tuple[int, float, np.ndarray]:
    """
    在 GPU 上计算余弦相似度。
    对比：sklearn CPU cosine_similarity 约 60ms，此函数约 0.03ms。
    """
    q = F.normalize(query_feat,   dim=-1)   # (1, D)
    g = F.normalize(gallery_feat, dim=-1)   # (N, D)

    sim_matrix = (q @ g.T)                  # (1, N)，GPU 上完成

    best_idx   = int(sim_matrix.argmax())
    best_score = float(sim_matrix[0, best_idx])

    return best_idx, best_score, sim_matrix.cpu().numpy()
```

### 5.3 完整 Warm Run 流水线

```python
def compute_similarity_optimized(
    query_arr:   np.ndarray,
    gallery_arr: np.ndarray,
    backbone:    torch.nn.Module,
    device:      torch.device,
    cache:       TwoLevelGalleryCache,
    pad_to_batch: int = 64,
    pooling:      str = "gap+cls",
) -> tuple[int, float, np.ndarray]:
    """
    生产级推理函数，整合所有优化。
    Warm Run 目标：< 50ms（L20）
    """
    # Step 1: GPU 预处理（uint8 H2D，~1.6ms）
    query_tensor = preprocess_on_gpu(query_arr, device)

    # Step 2: Query padding 推理（batch=64，~1ms）
    query_feat = infer_single_with_padding(
        backbone, query_tensor, pad_to_batch, device, pooling
    )

    # Step 3: Gallery 特征（L1/L2 缓存命中，~0ms）
    gallery_feat = cache.get_or_build(
        gallery_arr,
        build_fn=lambda arr: _extract_gallery_features(arr, backbone, device),
        device=device,
    )

    # Step 4: GPU 余弦相似度（~0.03ms）
    best_idx, best_score, sim_matrix = compute_similarity_gpu(
        query_feat, gallery_feat
    )

    return best_idx, best_score, sim_matrix
```
